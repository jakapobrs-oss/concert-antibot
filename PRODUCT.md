# Product

## Register

product

## Users

- **แฟนเพลงชาวไทย (ผู้ซื้อบัตร)** — เปิดเว็บช่วงนาทีก่อนบัตรเปิดขาย ใจเต้นแรง ใช้ได้ทั้งมือถือและโน้ตบุ๊ก งานหลักคือ เข้าคิว → เลือกที่นั่ง → โอนเงิน PromptPay + อัปสลิป → ได้ตั๋ว QR ภายในไม่กี่นาที ความเครียดสูง ทุกตัวเลข (ตำแหน่งคิว, เวลานับถอยหลัง, ราคา) ต้องอ่านได้ในแวบเดียว
- **ผู้ดูแลระบบ (admin)** — เปิดแดชบอร์ดดูยอดขาย คิว real-time และ bot log ระหว่างเปิดขายจริง ต้องการความหนาแน่นของข้อมูลและอ่านเร็ว
- **กรรมการสอบปริญญานิพนธ์** — เห็นเว็บผ่านโปรเจคเตอร์ตอน demo ระบบต้องดูเป็นงาน production จริง ไม่ใช่งานนักศึกษาทั่วไป

## Product Purpose

ระบบจองบัตรคอนเสิร์ตที่กันบอทและจัดคิวอย่างเป็นธรรม (โปรเจ็คจบ ป.ตรี วิทยาการคอมพิวเตอร์) — จุดต่างคือ fairness queue แบบสุ่ม + ยืนยันการจ่ายเงินจริงผ่านสลิป PromptPay ความสำเร็จ = ผู้ใช้จริงจองได้ลื่นจนจบ flow และกรรมการเชื่อว่าระบบนี้ขายจริงได้

## Brand Personality

**คืนแสดงสด · แม่นยำ · มีชีวิต** — ความรู้สึกยืนหน้าเวทีตอนไฟใกล้เปิด: มืด ร้อน มีพลัง แต่ทุกอย่างที่เป็น "ระบบ" (คิว ตัวเลข เวลา) คมและเที่ยงตรงเหมือนป้าย LED หน้างาน เสียงของแบรนด์จริงใจ ตรงไปตรงมา ไม่ขายฝัน (เราสัญญาเรื่อง "ความเป็นธรรม" ได้เพราะอธิบายกลไกได้จริง)

## Anti-references

- **เว็บ template SaaS/startup** — การ์ดหน้าตาเดียวกันเรียงแถว, gradient text, hero-metric, ลายเซ็น AI ทั่วไป
- **มืดจนอ่านยาก** — ห้ามเทาจางบนดำ ตัวหนังสือทุกจุดต้องรอด WCAG AA และอ่านชัดบนโปรเจคเตอร์สว่างต่ำ
- **ทางการจนจืด** — นี่คือเว็บคอนเสิร์ต ต้องมี motion มีลูกเล่น มีจังหวะ
- **เด็ก/การ์ตูน** — ไม่ใช้อีโมจิเป็น UI หลัก ไม่ใช้สีลูกกวาด ต้องดูเป็นระบบขายจริง

## Design Principles

1. **Stage, not dashboard** — หน้า public ใช้พลังของ "เวทีกลางคืน" เต็มที่ (แสง สี motion) แต่หน้า flow ที่ผู้ใช้กำลังลุ้น (คิว, ที่นั่ง, จ่ายเงิน) ความชัดเจนมาก่อนความสวยเสมอ
2. **ตัวเลขคือพระเอก** — ตำแหน่งคิว เวลานับถอยหลัง ราคา ใช้ฟอนต์ display (Chakra Petch) ขนาดใหญ่ สไตล์ป้าย LED ให้อ่านข้ามห้องได้
3. **แสงไฟเวที 2 ดวง** — แดง scarlet = action/แบรนด์, ทองอำพัน = ไฮไลต์/ราคา/ตัวเลขคิว ใช้อย่างมีหน้าที่ ไม่ใช่ตกแต่งมั่ว
4. **Motion สื่อสถานะและความสด** — equalizer = กำลังขาย, glow pulse = ที่นั่งที่เลือก, marquee = งานที่เปิดขาย ทุก animation มี reduced-motion fallback
5. **ความเป็นธรรมต้องมองเห็น** — กลไก anti-bot/คิวสุ่ม ถูกเล่าบนหน้าเว็บอย่างโปร่งใส เป็นจุดขายของ thesis

## Accessibility & Inclusion

- WCAG 2.1 AA: ข้อความปกติ ≥4.5:1, ข้อความใหญ่ ≥3:1 — เข้มงวดเป็นพิเศษเพราะต้องฉายโปรเจคเตอร์
- `prefers-reduced-motion: reduce` ปิด motion ทั้งหมด (มีอยู่แล้ว ต้องคงไว้)
- focus-visible ring ชัดทุก interactive element, ปุ่มที่นั่งมี title/aria
- ภาษาไทยเป็นหลัก (lang="th"), ฟอนต์ต้องรองรับไทย + Latin เต็มน้ำหนัก
